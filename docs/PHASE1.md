## Phase 1 路线图：最小 VM

**目标**：一个能跑纯计算程序的字节码解释器。

### 1.1 类型与值系统 ✅

- `Value` 枚举：`Int(i64)`, `Float(f64)`, `Bool(bool)`, `Null`
- 类型检查与转换辅助方法
- 算术/比较运算实现

### 1.2 字节码指令集 ✅

| 类别 | 指令 |
|---|---|
| 栈操作 | `Push`, `Pop`, `Dup` |
| 算术 | `Add`, `Sub`, `Mul`, `Div`, `Mod`, `Neg` |
| 比较 | `Eq`, `Neq`, `Lt`, `Gt`, `Lte`, `Gte` |
| 逻辑 | `And`, `Or`, `Not` |
| 控制流 | `Jump`, `JumpIfTrue`, `JumpIfFalse` |
| 变量 | `Load(addr)`, `Store(addr)` |
| 函数 | `Call(idx)`, `Return`, `Halt` |

### 1.3 虚拟机核心 ✅

- `VM` 结构：操作数栈、调用帧栈、指令指针
- `CallFrame`：函数引用、返回地址、局部变量数组
- 主循环：fetch -> decode -> execute，用 Rust match 做指令分发
- 代码存储：`Vec<Function>`，用索引引用函数

### 1.4 函数模型 ✅

- `Function` 结构：名称、参数数量、字节码块、局部变量数量
- `Call`：压入新帧，初始化局部变量为 `Null`
- `Return`：弹出当前帧，返回值压入操作数栈

### 1.5 标记-清除 GC

- 托管堆：`Vec<HeapObject>`，存储字符串等引用类型
- 根扫描：遍历所有栈帧的局部变量 + 操作数栈 + 全局变量
- Mark 阶段：从根出发，递归标记可达对象
- Sweep 阶段：回收未标记对象

### 1.6 字节码构造与二进制格式 ✅

- `Chunk`：字节码构建器，链式 `emit` 接口
- 二进制序列化格式
- 反序列化 / 验证

### 1.7 测试与演示

- [ ] 斐波那契数列（递归）
- [ ] 阶乘（递归）
- [ ] 带循环的求和
- [ ] 条件分支（if/else 等价物）
- [ ] 嵌套函数调用

**依赖顺序**：1.1 -> 1.2 -> 1.3 -> 1.4 -> 1.6 -> 1.7（主链），1.5 在 1.3 之后可并行。
